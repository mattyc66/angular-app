Architecture and Storage

The app is built with Electron and React, using IPC handlers in the main process to bridge the renderer with the file system, since a browser sandbox has no direct disk access. Comic data is persisted as JSON, and cover images were originally stored as base64 strings inside that same file. This was later switched to storing images as plain files on disk, referenced by path instead. The change reduced the size of the JSON file considerably, sped up initial load time, and removed a noticeable delay that base64 decoding was causing when applying sidebar filters.

Adding and Editing Comics

New comics are added through a form capturing title, status, chapter progress, and a cover image. Each comic is assigned a unique ID using Date.now(), and its cover image is renamed to match that ID plus its original file extension. Only the ID and extension are stored in the JSON entry, with the actual image kept in a dedicated cover images folder, keeping the data file lightweight and the images easy to locate on disk.

Multiple Links and Primary Sources

Comics often have more than one place to read them, so each entry stores its links as a nested array rather than a single value. One link can be flagged as primary, shown with a star icon in the edit form that toggles which link holds that status. The read button on each comic card uses whichever link is currently marked primary, so switching source sites is as simple as clicking the star next to a different link.

Search, Filtering, and UI performance

With close to a hundred comics tracked, several performance considerations went into the interface. The search bar is debounced to stop it re-filtering on every keystroke, which was causing noticeable lag. The library list itself uses @tanstack/react-virtual to render only the cards currently in view, keeping scrolling smooth regardless of collection size. Quick actions like editing or deleting a comic are handled through modals rather than navigating away, avoiding unnecessary reloads of the full list.

Import and Export

The app supports backing up and transferring a library using adm-zip. Exporting builds a comics-export.zip file containing the cover images and JSON data for whichever comics are currently visible under the active filters, rather than the entire library. Importing reads a zip in the same format and merges its contents into the app's existing JSON data and cover images folder, making it straightforward to move a library between machines or restore from a backup.
