Introduction to the library loan system

The Library Loan System project was developed to simplify the management of library resources such as books and DVDs. Using Java and JavaFX, the system supports various loanable items, tracks availability, and manages loans and returns. Additional features include user authentication, overdue fine calculation, item searches, and detailed item reports. With an intuitive interface and object-oriented design principles, the application is tailored for internal library employees, ensuring ease of use and functionality.

Design, Database, and Tooling

The brief specified that the design include object-oriented principles, a database, and a GUI built with JavaFX or AWT/Swing. The system was developed in IntelliJ using JavaFX for the interface, with the Gluon Scene Builder plugin integrated directly into the IDE, allowing FXML files to be opened in Scene Builder with a right-click. Books and DVDs were the two loanable item types chosen, each represented by shared reusable model and controller classes handling the item's ID, name, creator, availability, and image path, with a MySQL database storing item stock, loan records, and user accounts.

Login and Registration

On launch, users are prompted to either log in or register a new account. Registration allows a user to select between a standard or admin account type, though new accounts require admin approval before they can be used, satisfying the brief's requirement for controlled account creation. Once logged in, the login window is replaced by the main system view.

Main Interface and Navigation

The main view uses a BorderPane layout, with a sidebar on the left and the active page content on the right. The sidebar, labelled BookWise, provides navigation between Home, Loaned Items, Accounts, Sign Out, and Exit, swapping out the right-hand side of the BorderPane depending on the selected page. The home view uses tab panes to separate books and DVDs, each containing a search bar and a table listing the relevant items, books showing ID, name, and author, DVDs showing ID, title, and director, alongside options to view an item's full details or print them to a file.

Item Details and Loaning

Selecting an item and clicking "View" opens an item view page displaying its cover image alongside its details, including ID, name or title, creator, stock, and current availability. A customer name can be entered here to loan the item directly. Printing an item's details instead writes its full information to a text file, satisfying the brief's requirement to output item details, current availability, items on loan, and the earliest return date among any loaned copies.

Loaned Items and Overdue Fines

The Loaned Items page lists every item currently on loan in a searchable table, showing the item name, ID, customer name, product type, return date, and any late fee owed. A return button allows staff to process a return directly from this table, with support for adjusting the return date to calculate fines for items returned late.

Account Management

The Accounts page is restricted to admin users, prompting for admin login if a standard account attempts to access it. It displays a table of registering accounts, including their username, ID, and requested account type, with buttons allowing an admin to approve or deny each registration, completing the approval workflow required by the brief.
