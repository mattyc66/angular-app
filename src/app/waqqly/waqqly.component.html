What is Waqqly

Waqq.ly is a cloud-based web application designed to connect dog walkers with local dog owners. The platform enables dog owners to find reliable dog walkers in their area. The application provides a seamless user experience, using a map interface to display the locations of dog walkers. Dog walkers can also create profiles with relevant information, making it easier for dog owners to find trustworthy walkers. The application was built with scalability and flexibility in mind, using cloud technologies to ensure high performance even as the user base grows. It utilizes Firebase for authentication, real-time database management, and storage, while also integrating various third-party APIs for additional functionality. The app provides essential features such as user authentication, map tracking of walkers, and secure storage of user data and images. Hosted on AWS, Waqq.ly ensures that the platform can scale with demand, offering a reliable and efficient solution for dog owners and walkers.

Meeting the Requirements

Several decisions were made specifically to satisfy the brief's non-functional requirements. Firebase was chosen for authentication and data storage partly to reduce infrastructure management overhead, though this came with an inherent trade-off around vendor lock-in, weighed against the development speed it offered. Containerising the geocoding microservice with Docker addressed portability and virtualisation directly, since the container could be deployed to Google Cloud Run independently of the rest of the stack, rather than being tied to a specific server environment. Cloud Run's autoscaling also addressed resource management and cost control together, since it scales down automatically when the service isn't in use rather than requiring a constantly running server. Reusability was considered in the frontend through shared React components, such as the navigation bar, used consistently across pages.

Containerisation and the Geocoding Microservice

Locating dog walkers required converting a user's postcode into map coordinates. This was implemented as a standalone microservice using Express, wrapping a fetchCoordinates function that queries the Google Maps API. The microservice was containerised using Docker and deployed to Google Cloud Run, which handles autoscaling and integrates easily with other Google Cloud services. Separating this functionality into its own containerised service, rather than folding it into the main app, kept it independently deployable and scalable regardless of load on the rest of the application. On the frontend, the React Leaflet library was used to plot walker locations on an interactive map.

Authentication and NoSQL Database

User authentication and data storage were handled through Firebase, chosen as a NoSQL, cloud-based solution suited to the app's relatively simple, document-like data (user profiles, postcodes, and descriptions). Firebase Authentication manages user credentials and login, while the Real-Time Database stores profile information and Firebase Storage holds profile images, allowing users to log in and retrieve their data with minimal custom backend code.

Deployment

The React frontend was version-controlled in a GitHub repository connected to AWS Amplify, which automatically redeployed the app whenever new changes were pushed. Combined with the containerised microservice on Cloud Run, this gave the application two independently deployable parts, each scaling and updating without affecting the other.
